# 软删除采用正交状态字段，而非 QuestCategory 新增枚举值

删除任务需要标记任务的“已删除”状态，同时保留其原有的主线/支线/每日归属以便恢复。若把“已删除”做成 `QuestCategory` 的第四个枚举值，恢复时需要额外记录“删除前的分类”，且所有按 `category` 做的分组、筛选逻辑都要专门排除这个值，容易遗漏。因此选择在 `Quest` 上新增正交字段 `deletedAt: Date?`，删除与恢复只是置位/清空该字段，`category` 全程不变；这与项目中 `isCompleted`（完成状态）已经采用的正交状态模式保持一致。

代价是后续所有从 `quests` 派生列表的入口（`listItems`、重复任务视图等）都必须显式过滤 `deletedAt != nil` 的任务，需要在实现时统一收口，避免遗漏导致已删除任务泄露到常规列表。
