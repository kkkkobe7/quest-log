# 无截止时间任务默认延续设计

## 背景

QuestList 当前普通任务的 `displayDate` 规则是：有截止时间使用 `dueDate`，无截止时间使用固定的 `startDate`。因此，一个没有截止时间且昨天创建的任务，如果今天仍未完成，今天不会出现在「全部任务 → 当日」视图中。用户期望这类任务像待办事项一样默认延续：只要未完成，就继续出现在今天，直到完成。

## 目标

- 无截止时间且未完成的普通任务，在开始日期过后自动滚到「今天」。
- 未来开始的无截止时间任务仍保留在未来开始日期，不提前显示。
- 无截止时间任务完成后停止延续，并在「已完成任务」中按完成日期分组。
- 不修改用户原始 `startDate`，保留任务最早计划/开始时间。
- 保持有截止时间任务、重复任务 occurrence、已删除任务详情等既有行为不回归。

## 非目标

- 不引入“逾期任务”独立分区。
- 不改变有截止时间任务的筛选与分组语义。
- 不改变重复任务 occurrence 生成规则。
- 不新增自定义延期策略或手动顺延按钮。
- 不改变 XP 发放规则；已发放过 XP 的任务重新完成时不重复加 XP。

## 核心语义

### 有效显示日期

Core 层新增统一的「有效显示日期」概念，供筛选、排序和分组共同使用：

- 有 `dueDate`：有效显示日期为 `dueDate`。
- 无 `dueDate` 且未完成：
  - `startDate` 在未来：有效显示日期为 `startDate`。
  - `startDate` 在今天或过去：有效显示日期滚到今天；日期部分使用今天，时间部分沿用 `startDate` 的时分秒。
- 无 `dueDate` 且已完成：有效显示日期为 `completedAt`。
- 无 `dueDate`、已完成但历史数据缺少 `completedAt`：有效显示日期回退为 `startDate`。
- 重复任务 occurrence：仍按 occurrence 自身日期，不受本规则影响。

### 完成与取消完成

- 普通无截止时间任务完成后停止延续。
- 取消完成后，该任务重新按未完成状态滚到「今天」。
- 再次完成时，`completedAt` 更新为最新完成时间。
- 再次完成时如果 XP 已发放，不重复增加 XP。

## 实现方案

### Core 统一计算

在 Core 层提供根据 `now` 与 `calendar` 计算有效显示日期的方法。`QuestStore.listItems(in:now:calendar:)` 对普通任务使用该有效显示日期进行筛选，而不是直接使用固定的 `Quest.displayDate`。

`QuestListItem` 保存 Core 计算好的 `displayDate`，确保「全部任务」「已完成任务」「目标视图」中的排序、日期分组和筛选使用同一个结果，避免 UI 层各自推导导致不一致。

### 数据兼容性

不新增持久化字段，不修改现有 JSON 数据结构。`startDate` 仍保留原始开始/创建语义；滚动只发生在读取和列表投影阶段。

历史已完成任务如果缺少 `completedAt`，按 `startDate` 分组，避免因为旧数据缺失而从列表中消失。

### UI 影响

- 「全部任务」：无截止时间的过期未完成任务显示在今天。
- 「未来」：今天滚动的任务不进入未来；未来开始的任务仍进入未来。
- 「已完成任务」：无截止时间任务按完成日期筛选与分组。
- 「重复任务」：继续使用 occurrence 日期和父任务概览规则，不改变。
- 「已删除任务」：继续按删除时间倒序，详情访问控制不改变。

## 测试设计

在 `QuestListCoreChecks` 中覆盖：

- 昨天创建、无截止时间、未完成任务，在今天出现在「当日」。
- 今天创建、无截止时间、未完成任务，在今天正常出现。
- 明天开始、无截止时间、未完成任务，今天不出现，「未来」出现。
- 昨天创建、昨天已完成任务，今天不出现在「全部任务 → 当日」，并在「已完成任务」中按完成日期筛选和分组。
- 取消完成后，任务重新滚回「全部任务 → 当日」。
- 再次完成后，`completedAt` 更新为最新完成时间，XP 不重复发放。
- 有截止时间任务仍按 `dueDate`。
- 重复任务 occurrence 仍按 occurrence 日期。

## 验收标准

- 用户昨天创建且未完成的无截止时间任务，今天打开 App 后出现在「全部任务 → 当日 → 今天」。
- 完成该任务后，明天它不再滚到今天，而是在「已完成任务」中按完成日期展示。
- 原始 `startDate` 不被每日滚动覆盖。
- `QuestListCoreChecks` 通过。
- `swift build` 通过。
- 代码审查无高信号问题。
